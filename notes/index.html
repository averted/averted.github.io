DOM Query Params (URLSearchParams)

2018-11-06
:: tags:  #dom  #js

Extract query string params (replace qs?)

Assuming current URL is: ?post=1234&action=edit


var urlParams = new URLSearchParams(window.location.search);

console.log(urlParams.has('post')); // true
console.log(urlParams.get('action')); // "edit"
console.log(urlParams.getAll('action')); // ["edit"]
console.log(urlParams.toString()); // "?post=1234&action=edit"
console.log(urlParams.append('active', '1')); // "?post=1234&action=edit&active=1"

:: tags:  #ts

Polymorphism (Generics)

type Filter = {
  <T>(array: T[], fn: (item: T) => boolean): T[]
}

let filter: Filter = (arr, f) => { ... }

// T is bound to number
filter([1,2,3], (item) => item > 2)

The place where you declare a generic type dictates when TS will bind a concrete type to your generic. Because we declared as part of call signature, TS will bind a concrete type to T when we actually call filter.

Index Signature

The [key: T]: U is called an index signature, and is the way you tell TS that an object might contain more keys.

let a: {
  b: number
  c?: string
  [key: string]: boolean
}

  (from: Date, to: Date, destination: string): Reservation
}

  destination?: string
